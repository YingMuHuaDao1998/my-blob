# query 技能

## 核心职责

在 LLM Wiki 中检索相关知识，综合提炼后给出回答，并附上双链引用。

## 触发方式

- 用户执行 `/query <问题>`
- 用户直接提问 → 隐式触发

## 检索流程

### 步骤 1：读取索引

先读 `wiki/index.md`，快速定位相关页面：

```bash
cat wiki/index.md
```

### 步骤 2：检索相关页面

优先检索路径（按优先级）：
1. `wiki/sources/` — 原始资料摘要
2. `wiki/concepts/` — 概念定义
3. `wiki/entities/` — 实体信息
4. `wiki/syntheses/` — 综合报告
5. `{Topic}/` — 已结构化的主题知识

### 步骤 3：读取相关页面内容

读取定位到的相关文件，提取与问题相关的段落。

### 步骤 4：综合回答

生成结构化回答：
- 回答核心问题
- 每个观点必须标注 `[[wikilink]]` 引用来源
- 如有冲突观点，对比呈现
- 给出知识来源层级（摘要层 / 概念层 / 原始主题层）

### 步骤 5：记录操作

追加到 `wiki/log.md`：

```markdown
## [YYYY-MM-DD] query | {问题简述}
- **检索**: [[Page1]], [[Page2]]
- **输出**: [[synthesis-slug]] 或直接回答
```

## 回答格式建议

```markdown
## 回答：{问题}

{综合回答内容}

## 参考来源
- [[wiki/来源页面]] — {简要说明}
- [[Topic/原始文档]] — {简要说明}
```

## 注意事项

- 优先从 `wiki/syntheses/` 和 `wiki/sources/` 读取提炼层，再追溯原始主题
- 回答必须包含双链引用，禁止空洞回答
- 不确定时明确说"知识库暂无相关内容，建议摄入 raw/ 后再检索"
