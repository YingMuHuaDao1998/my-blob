# lint 技能

## 核心职责

对 LLM Wiki 进行全局体检，发现问题并修复，保证知识网络的质量和连通性。

## 触发方式

- 用户执行 `/lint`
- 用户说"检查知识库健康" → 隐式触发

## 体检维度

### 1. 孤岛页面检查

没有任何双链引用的页面（死链接入点）：

```bash
# 列出 wiki/ 下所有 .md 文件
# 检查每个文件是否被其他页面引用
```

### 2. 死链检查

存在指向不存在页面的 `[[wikilink]]`：

```bash
# 提取所有 wiki/ 下文件中的 [[xxx]] 链接
# 对比 index.md 中是否存在这些页面
```

### 3. index.md 同步检查

index.md 中记录的页面是否都实际存在：

```bash
# 提取 index.md 中所有 [[page]] 引用
# 验证每个引用的页面文件是否存在
```

### 4. 矛盾检查

同一概念出现多个说法且未标注冲突：

```markdown
# 读取同概念多个页面
# 检查是否都有 ## 知识冲突 区块
```

### 5. 孤儿文件检查

`raw/` 下有文件未被 ingest 过，也不在 archive 中：

```bash
find raw/ -type f \( -name "*.md" -o -name "*.pdf" \) | grep -v "09-archive"
```

## 输出格式

检查完成后，输出：

```markdown
## 知识库体检报告

### 孤岛页面（X个）
- [[孤岛页面1]] — 无任何引用

### 死链（X个）
- [[某页面]] → 指向不存在的 [[TargetPage]]

### index 不同步（X个）
- index 记录了 [[MissingPage]]，但文件不存在

### 孤儿文件（X个）
- raw/01-articles/xxx.md — 未被 ingest

### 冲突待确认（X个）
- [[概念A]] 有两个不同说法，见各页面 ## 知识冲突
```

## 自动修复范围

| 问题类型 | 是否自动修复 |
|----------|------------|
| 孤岛页面 | ✅ 追加 `## 关联连接` 或提示用户补充 |
| 死链 | ✅ 删除死链引用 |
| index 不同步 | ✅ 删除 index 中不存在的记录 |
| 孤儿文件 | ❌ 仅报告，由用户决定是否 ingest |
| 矛盾冲突 | ❌ 报告给用户，等待确认处理方式 |

## 记录操作

每次 lint 后追加 log：

```markdown
## [YYYY-MM-DD] lint | 知识库体检
- **孤岛**: X个（已修复Y个，剩余Z个待处理）
- **死链**: X个（已修复Y个）
- **index不同步**: X个（已修复）
- **未 ingest**: X个（请执行 /ingest）
```
