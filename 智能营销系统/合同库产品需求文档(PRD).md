## 一、产品概述

### 1.1 产品背景

智能营销系统旨在帮助企业实现营销全流程数字化管理。合同作为企业与客户合作的法律凭证，是营销链条中的关键数据资产。当前企业面临以下痛点：

| 痛点 | 影响 |
|------|------|
| 合同数据分散 | 散落在各业务系统、Excel、纸质文件中，难以统一管理 |
| 录入效率低下 | 人工逐字段录入，耗时长、易出错 |
| 检索困难 | 缺乏有效的检索手段，找合同费时费力 |
| 数据利用率低 | 合同数据无法支撑营销决策分析 |

### 1.2 产品定位

合同库是智能营销系统的**核心数据基础设施**，定位为：

> **企业合同数据的统一入口**——集中存储、智能解析、便捷检索，为营销决策提供数据支撑。

### 1.3 产品目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 效率目标 | 提升合同录入效率，减少人工录入工作量 | 录入时间缩短 60% 以上 |
| 数据目标 | 实现近三年合同数据统一存储，形成数据资产 | 合同数据入库率 ≥ 95% |
| 体验目标 | 提升合同检索效率，快速定位目标合同 | 检索时间 < 10 秒 |

### 1.4 成功指标（KPI）

| 指标 | 目标值 | 衡量周期 |
|------|--------|----------|
| 合同录入时间 | 较原方式缩短 ≥60% | 上线后3个月 |
| OCR识别准确率 | ≥95% | 上线后1个月 |
| 用户检索成功率 | ≥90% | 上线后2个月 |
| 系统活跃使用率 | ≥80% 合同数据入库 | 上线后6个月 |

## 二、目标用户

### 2.1 用户画像

| 用户角色 | 职责 | 核心诉求 | 使用频率 |
|----------|------|----------|----------|
| **销售经理** | 负责客户签约，录入合同 | 快速录入、快速查历史合同 | 每周 5-10 次 |
| **销售助理** | 合同档案整理、数据录入 | 批量处理、减少重复劳动 | 每天 5-20 次 |
| **财务人员** | 合同审核、付款跟进 | 准确的金额和付款信息 | 每天 3-10 次 |
| **管理层** | 查看合同数据、决策分析 | 数据统计、趋势分析 | 每周 2-5 次 |

### 2.2 核心用户场景

#### 场景一：新签合同录入

> **角色**：销售经理
> **场景**：刚与客户完成签约，需要将合同信息录入系统
> **期望**：快速完成录入，不想逐字段手填

**当前痛点**：
- 需要手动填写 20+ 个字段
- 容易填错，需要反复检查
- 合同附件要单独上传到其他系统

**目标体验**：
- 上传合同扫描件 → OCR 自动识别填充 → 快速确认保存

#### 场景二：查找历史合同

> **角色**：销售经理
> **场景**：客户询问历史签约情况，需要快速找到 2 年前的合同
> **期望**：几秒内找到目标合同

**当前痛点**：
- 合同分散存储，不知道在哪
- 即使有电子版，也难以快速定位
- 按客户名搜索，经常搜不到

**目标体验**：
- 输入客户名（模糊匹配）→ 快速筛选 → 查看合同详情

#### 场景三：批量导入历史合同

> **角色**：销售助理
> **场景**：公司要求将过去 3 年的合同全部录入新系统
> **期望**：批量处理，不要一份份手动录入

**当前痛点**：
- 几百份合同，手动录入工作量巨大
- 格式不统一，处理困难

**目标体验**：
- 批量上传文档 → 系统自动解析 → 人工校验确认

## 四、功能详细设计

### 4.1 合同录入

#### 4.1.1 手动录入

**用户价值**：销售经理签订合同后，快速将合同信息录入系统，建立数据资产。

**用户故事**：
> 作为销售经理，我希望能够手动录入合同信息，以便将签约数据保存到系统中。

**功能要点**：

| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| 表单录入 | 提供合同信息录入表单 | P0 |
| 字段校验 | 金额格式、日期范围、必填校验 | P0 |
| 附件上传 | 支持上传合同扫描件、补充协议 | P1 |
| 表单暂存 | 支持保存草稿，后续继续填写 | P2 |

**关键字段**：

| 字段名 | 字段类型 | 是否必填 | 校验规则 |
|--------|----------|----------|----------|
| 客户名称 | 文本 | 是 | 长度 ≤100 字符 |
| 合同金额 | 数值 | 是 | ≥0，小数点后两位 |
| 签约日期 | 日期 | 是 | ≤当前日期 |
| 合同有效期起 | 日期 | 是 | - |
| 合同有效期止 | 日期 | 是 | ≥有效期起 |
| 合同附件 | 文件 | 否 | PDF/Word/图片，≤50MB |

**交互设计要点**：
- 表单分步引导：基本信息 → 付款信息 → 附件上传
- 智能联想：客户名称输入时，联想历史客户
- 错误提示：实时校验，错误信息明确

#### 4.1.2 批量导入

**用户价值**：销售助理可批量处理大量历史合同，大幅提升录入效率。

**用户故事**：
> 作为销售助理，我希望能够批量上传合同文档，由系统自动解析，减少手动录入工作量。

**功能要点**：

| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| 批量上传 | 支持一次上传多个合同文档 | P1 |
| 格式支持 | 支持 PDF、Word、图片格式 | P1 |
| 进度反馈 | 显示解析进度和状态 | P1 |
| 异常处理 | 标识解析失败的文档，支持重试 | P1 |

**交互流程**：
1. 用户选择多个文档上传
2. 系统逐个解析，显示进度
3. 解析完成后，展示结果列表
4. 用户确认或修正解析结果
5. 批量保存到数据库

### 4.2 OCR 智能解析

#### 4.2.1 功能概述

**用户价值**：通过 AI 技术自动识别合同内容，将录入时间从 10 分钟缩短到 2 分钟以内。

**用户故事**：
> 作为销售经理，我希望上传合同后系统能自动识别内容，我只需确认和微调，不用逐字段手填。

**技术指标**：

| 指标 | 目标值 |
|------|--------|
| 支持格式 | PDF、Word、JPG、PNG |
| 识别准确率 | ≥95% |
| 单文档解析时间 | ≤10秒 |
| 覆盖字段数 | 20+ 核心字段 |

**识别字段清单**：

| 字段 | 说明 | 识别优先级 |
|------|------|------------|
| 客户名称 | 签约对方企业名称 | 高 |
| 合同金额 | 合同总金额 | 高 |
| 签约日期 | 合同签订日期 | 高 |
| 有效期起止 | 合同执行期限 | 高 |
| 付款条款 | 付款节点和比例 | 中 |
| 违约责任 | 违约相关条款 | 中 |
| 签约主体 | 本方签约公司 | 中 |
| 合同编号 | 合同唯一编号 | 中 |

**异常处理机制**：
- 置信度低于阈值 → 标记为「待确认」，高亮显示
- 无法识别 → 标记为「需人工填写」，提示用户录入

#### 4.2.2 交互流程

```
用户上传文档 → OCR解析 → 展示识别结果
                           ↓
                    用户确认/修正
                           ↓
                    保存到数据库
```

### 4.3 标签管理

**用户价值**：为合同打上多维标签，便于后续分类检索和数据分析。

**用户故事**：
> 作为销售经理，我希望能为合同打上行业、项目类型等标签，方便后续筛选和管理。

**标签体系**：

| 标签维度 | 标签示例 | 场景说明 |
|----------|----------|----------|
| 行业 | 教育、医疗、政府、金融 | 按客户行业分类 |
| 项目类型 | 新签、续约、增补 | 按合同性质分类 |
| 区域 | 华东、华南、华北、西南 | 按客户地域分类 |
| 自定义 | 用户自定义内容 | 灵活扩展 |

**功能要点**：

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 打标签 | 为单个/批量合同添加标签 | P1 |
| 标签筛选 | 按标签组合筛选合同 | P1 |
| 标签管理 | 创建、编辑、删除标签 | P1 |
| 标签统计 | 各标签下合同数量统计 | P2 |

### 4.4 数据存储

**设计原则**：
- 近三年合同数据在线存储，支持实时查询
- 超过三年的数据进行归档处理
- 附件存储采用对象存储，降低存储成本

**存储容量估算**：

| 数据类型 | 单条大小 | 预估数量 | 总容量 |
|----------|----------|----------|--------|
| 合同结构化数据 | ~2KB | 100,000 条 | ~200MB |
| 合同附件 | ~2MB | 100,000 份 | ~200GB |

---

### 4.5 合同检索

**用户价值**：快速找到目标合同，节省检索时间。

**用户故事**：
> 作为销售经理，我希望能通过客户名、金额范围等条件快速找到目标合同。

**检索维度**：

| 条件 | 检索方式 | 使用场景 |
|------|----------|----------|
| 客户名称 | 模糊匹配 | 记不清全名时快速定位 |
| 合同金额 | 区间筛选 | 查找大额合同 |
| 签约日期 | 时间范围 | 查找某时间段合同 |
| 合同状态 | 下拉选择 | 筛选待审批/已生效/已终止 |
| 标签 | 多选组合 | 按分类查找 |

**高级功能**：

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 全文检索 | 搜索合同内容、备注等 | P2 |
| 条件保存 | 保存常用检索条件 | P2 |
| 结果导出 | 导出检索结果为 Excel | P1 |
