## 一、概述

### 1.1 文档目的

本文档定义智能营销系统中「合同库」模块的功能需求，用于指导开发团队进行系统设计与开发。

### 1.2 系统背景

合同库是智能营销系统的核心数据管理模块，负责合同数据的录入、存储、解析、检索与标签管理，为后续营销分析、客户管理等功能提供数据基础。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| 合同库 | 系统中统一存储和管理合同数据的功能模块 |
| OCR | 光学字符识别，用于自动识别合同文档中的文字内容 |
| 结构化解析 | 将非结构化的合同文档转换为结构化数据的过程 |

### 2.2 F001 合同录入

#### 2.2.1 功能描述

支持合同的录入功能，包括手动录入和批量导入两种方式。

#### 2.2.2 功能要求

**手动录入：**
- 提供字段级校验功能
  - 金额格式校验
  - 日期范围校验
  - 必填字段校验
- 支持附件上传
  - 合同扫描件
  - 补充协议
  - 支持常见文件格式（PDF、Word、图片等）

**批量导入：**
- 支持批量上传合同文档电子版
- 支持常见文档格式（PDF、Word、图片）
- 提供导入进度反馈
- 支持导入失败记录导出

#### 2.2.3 验收标准

| 编号 | 验收标准 |
|------|----------|
| F001-1 | 手动录入时，金额字段仅接受数字和小数点，且不超过两位小数 |
| F001-2 | 手动录入时，日期字段校验逻辑正确（如结束日期≥开始日期） |
| F001-3 | 附件上传支持 PDF/Word/JPG/PNG 格式，单个文件≤50MB |
| F001-4 | 批量导入支持一次性上传≥100个文档 |
| F001-5 | 导入失败时，显示具体失败原因并提供重试机制 |

### 2.3 F002 合同结构化解析

#### 2.3.1 功能描述

通过 OCR 技术自动识别合同文档中的关键字段，实现合同内容的结构化，减少人工录入工作量。

#### 2.3.2 功能要求

**识别范围：**
- 覆盖 20+ 核心字段，包括但不限于：
  - 客户名称
  - 合同金额
  - 有效期（起始日期、终止日期）
  - 付款节点
  - 违约责任
  - 签约主体
  - 签约时间
  - 合同编号

**技术要求：**
- 支持格式：PDF、Word、图片（JPG/PNG等）
- 结构化准确率：≥95%
- 异常字段自动标记，待人工确认

**结果应用：**
- 解析后数据自动映射至合同数据库对应字段
- 支持与历史合同数据关联比对
- 支持手动编辑修改解析结果

#### 2.3.3 验收标准

| 编号 | 验收标准 |
|------|----------|
| F002-1 | OCR识别准确率≥95%（以抽样测试为准） |
| F002-2 | 异常/低置信度字段自动标记为"待确认"状态 |
| F002-3 | 解析结果支持手动修改，修改后自动保存 |
| F002-4 | 单个合同文档解析时间≤10秒 |
| F002-5 | 支持识别后的数据与历史合同自动关联提示 |

### 2.4 F003 合同标签管理

#### 2.4.1 功能描述

为合同数据建立多维标签体系，支持灵活的分类与筛选。

#### 2.4.2 功能要求

**标签体系维度：**

| 维度 | 标签示例 | 说明 |
|------|----------|------|
| 行业标签 | 教育、医疗、政府、金融、零售等 | 按客户所属行业分类 |
| 项目类型标签 | 新签、续约、增补等 | 按合同类型分类 |
| 区域标签 | 华东、华南、华北、西南等 | 按客户地域分类 |
| 自定义标签 | 用户自定义内容 | 支持用户创建个性化标签 |

**标签功能：**
- 支持单条/批量打标签
- 支持标签的增删改查
- 支持标签合并与删除
- 支持标签使用统计分析

#### 2.4.3 验收标准

| 编号 | 验收标准 |
|------|----------|
| F003-1 | 单条合同支持添加≥10个标签 |
| F003-2 | 支持按标签组合筛选合同列表 |
| F003-3 | 标签支持层级结构（最多3级） |
| F003-4 | 标签命名规则：≤20字符，支持中英文数字 |

## 五、接口设计（概要）

### 5.1 合同管理接口

| 接口名称 | 方法 | 路径 | 说明 |
|----------|------|------|------|
| 创建合同 | POST | /api/contracts | 手动录入合同 |
| 批量导入 | POST | /api/contracts/import | 批量上传合同文档 |
| 合同列表 | GET | /api/contracts | 分页查询合同列表 |
| 合同详情 | GET | /api/contracts/{id} | 获取合同详情 |
| 更新合同 | PUT | /api/contracts/{id} | 更新合同信息 |
| 删除合同 | DELETE | /api/contracts/{id} | 删除合同（软删除） |

### 5.2 OCR 解析接口

| 接口名称 | 方法 | 路径 | 说明 |
|----------|------|------|------|
| 文档解析 | POST | /api/ocr/parse | 上传文档进行OCR解析 |
| 解析结果 | GET | /api/ocr/result/{taskId} | 获取解析结果 |

### 5.3 标签管理接口

| 接口名称 | 方法 | 路径 | 说明 |
|----------|------|------|------|
| 标签列表 | GET | /api/tags | 获取所有标签 |
| 创建标签 | POST | /api/tags | 创建新标签 |
| 合同打标签 | POST | /api/contracts/{id}/tags | 为合同添加标签 |
