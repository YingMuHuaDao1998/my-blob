1. keys

- keys  pattern

- 功能：查找所有给定模式pattern的 key，pattern为正则表达式

- 说明：keys 的速度非常快，但在一个大的数据库中使用它可能会阻塞当前服务器的服务，所以生产环境中一般不使用该命令，而使用 scan 命令代替

  

2. exists

- 格式：EXISTS  key

- 功能：检查给定 key是否存在

- 说明：若 key存在，返回 1，否则返回 0

  

3. rename

- 格式：rename key  newKey
- 功能：将 key 改为 newkey
- 说明：当 key 和 newkey 相同，或者 key 不存在时，返回一个错误。当 newkey已经存在时，rename命令将覆盖旧值，改名成功时ok，失败时候返回一个错误。



4. move

- 格式：move key db
- 功能：将当前数据库的 key 移动到指定的数据库 db 当中
- 说明：如果当前数据库（源数据库）和给定数据库（目标数据库）存在相同名字的给定 key，或者 key 不存在于当前数据库，那么 move 没有任何效果，移动成功返回 1，失败返回 0



5. type

- 格式：type key
- 功能：返回key 所存储的值的类型
- 说明：返回值有一下 6 种

none（key 不存在） string（字符串）     list（列表）

set（集合）                zset（有序集）        hash（哈希表）



6. expire

- 格式：expire key seconds
- 功能：为给定 key设置生存时间，当 key过期时（生存时间为0），它会被自动删除，expire的时间单位为秒，pexpire的时间单位为毫秒，在 redis 中，带有生存时间的key 被称为"易失的"(vollatile)
- 说明：生存时间成功设置为 1，若 key 不存在时返回 0，rename 操作不会改变 key 的生存空间。



7. ttl与pttl

- 格式：ttl key
- 功能：ttl  time to live 返回给定key 的剩余生存时间
- 说明：其返回值存在三种可能

当key 不存在时，返回-2

当key存在但没有设置剩余生存时间时，返回-1

否则，返回 key 的剩余生存时间，ttl 返回的剩余时间单位为秒，而pttl命令的返回的时间单位为毫秒



8. persist

- 格式：persist  key

- 功能：去除给定 key 的生存时间，将这个key 从"易失的"转换成"持久的"

- 说明：当生存时间转移成功时，返回 1,若 key 不存在或 key 没有设置生存时间，则返回 0

  

9. randomkey

- 格式：randomkey
- 功能：从当前数据库中随机返回（不删除）一个 key
- 说明：当数据库不为空时，返回一个 key，当数据库为空时，返回 nil

在生产环境中，尤其重要，可以判断数据库中数据是否为空，用 randomkey 而不是 keys *

因为 keys * 存在弊端

11. set

- 格式：set key value 【ex seconds | px milliseconds】【nx｜xx】

  set除了可以直接将 key 的值设为 value 外，还可以指定一些参数

  - ex seconds：为当前 key 设置过期时间，单位秒
  -  px milliseconds：为当前 key 设置过期时间，单位毫秒
  - nx：指定的 key 不存在才会设置成功，用于添加该指定的 key
  - xx：指定的 key 存在才会设置成功，用于更新指定key 的 value

```
set name value ex time 创建key 和 value 并设置过期时间
```

12. setex和psetex

- 格式：setex/psetex key seconds value

- 功能：set expire ，其不仅为key 指定了 value，还为其设置了生存时间，setex 的单位为秒，psetex 单位为毫秒。

- 说明：如果 key 已经存在，则覆写其值，该命令类似与一下两个命令，不同之处是，setex是一个原子性操作，关联值和设置生存时间两个动作会在同意时间完成，该命令在 redis用作缓存时，非常实用。

  set key value

  expire  key seconds 设置生存时间

  ```
  setex name 300 tom
  ```

13. setnx

- 格式：setnx key value

- 功能：set if not exists，将key 的值设为 value，当且仅当key 不存在，值能设置成功，若给定的 key已经存在，则 setnx不做任何动作，成功，返回 1，失败，返回 0.

- 说明：该命令等价于set key value nx

17. append

- 格式：append key value
- 功能：如果 key已经存在并且是一个字符串，append 命令将值追加到原来值的末尾，当 key 不存在时，append 就简单的将 key 设置为 value
- 说明：结果为追加后，字符串的长度。如果想在插入空格，得使用""的方式
